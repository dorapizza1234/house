## 한 줄 컨셉

집으로(Jibeuro)는 가족 구성원들이 **귀가 여부를 공유**하고, **프리셋 감사 메시지로 포인트**를 모으고, **가족 공용 식물을 함께 돌보며** 유대감을 강화하는 **가족 커뮤니티 웹앱**입니다.

## 오리진 스토리

아빠가 딸이 집에 있는지 모르겠다며 "미니소화기를 신발에 꽂아놓으라"고 한 일화에서 출발. GPS 자동 추적이 아닌 **"집 재실 여부 + 가족 공동체 강화"** 컨셉으로 발전.

## 프로젝트 목적

- **포트폴리오용** 신규 프로젝트 (딸어디가와는 별개)
- Spring Boot 심화 + React/TypeScript 신규 학습이 목표
- 3~4개월 Solo 개발

## 타깃 유저

- 핵가족 (부모 + 성인 자녀)
- 부모가 자녀의 귀가 여부를 궁금해하는 가정
- 가족 간 감사/사랑 표현이 서툰 가정
- Phase 1은 웹 우선 (데스크탑/모바일 웹), Phase 2는 네이티브 앱 확장 예정

## 해결하려는 문제

1. 부모가 자녀가 집에 있는지 없는지 모름 (한 집에 살아도 방이 떨어져 있으면)
2. 가족 간 감사 표현이 쑥스럽거나 어색해서 표현을 못 함
3. 가족 일정/기념일 관리가 각자 따로 흩어져 있음
4. 가족 간 정서적 유대감 강화 수단 부족

## Phase 1 MVP 스코프 (확정)

1. **회원가입/로그인** — JWT (Access 30분 + Refresh 7일), BCrypt 해시
2. **가족 그룹 생성 + 초대** — JWT 초대링크 (24h 만료, 1회용)
3. **가족 대시보드** — 집/밖 **수동** 토글 (위치검증 없음, 버튼만)
4. **가족 캘린더** — 생일/제사/일정, 매년반복 지원
5. **체크인 로그** — 도착/외출 시각 자동 기록
6. **귀가 시간 통계/예측** — Spring `@Scheduled` 배치, 요일별 평균 + 가중 이동평균 + 이상치 제거
7. **프리셋 메시지 + 포인트 시스템**
   - 메시지 종류: 감사 / 사랑 / 잘다녀와요 / 미안 / 힘내요
   - **프리셋만 포인트 지급, 자유입력 0점**
   - **하루 프리셋 메시지 3회 제한** (4번째부터 차단)
   - **이벤트일 배수는 곱셈 방식** (어버이날 ×2 + 엄마생신 ×2 = 최종 ×4 = 40점)
   - 이벤트 종류: 공휴일(어버이날/어린이날/스승의날/발렌타인) + 가족일정(매년반복=true)
   - **철회 기능 없음, 포인트 회수 없음**
8. **가족 공용 식물 키우기** — 가족당 1개, 물주기만, 24h → WILTED, 72h → DEAD

## Phase 1에서 제외 (Out of Scope)

- 위치 자동 추적 (Phase 2 네이티브 앱에서)
- 고양이/강아지 키우기 (Phase 2)
- 사진 일기 (Phase 3+)
- 투표/공지 (Phase 3+)
- Redis 캐싱 (MVP 완성 후 성능 개선 단계로 미룸)

## 기술 스택 (확정)

- **프론트엔드**: Next.js 14 + TypeScript + Tailwind CSS → **Vercel** 배포
- **백엔드**: Spring Boot 3 + Java 17 + JPA + Spring Security + JWT → **Railway** 배포
- **DB**: PostgreSQL → **Supabase** (무료 500MB)
- **배치**: Spring Boot 내부 `@Scheduled` + Java Stream API
  - Python, Node.js 배치 **사용 안 함** (solo dev에서 언어 분산 최소화)
- **인증**: JWT (Access 30분 + Refresh 7일) + BCrypt
- **CI/CD**: GitHub Actions + Vercel/Railway 자동 배포
- **도구**: VS Code (프론트) + IntelliJ IDEA Community (백엔드), DBeaver, Bruno

## 포인트 규칙 상세

- 프리셋 메시지: 1회 = 10점 (기본)
- 자유입력 메시지: 0점
- 이벤트 배수 (복수 이벤트 중첩 시 곱셈):
  - 어버이날에 부모님께 프리셋 메시지 → ×2
  - 어버이날 + 엄마 생신 겹친 날 → ×2 × ×2 = ×4 = 40점
- 하루 프리셋 3회까지만 포인트 지급, 4번째부터는 전송 자체 차단

- 프론트엔드 Lighthouse 점수 90+

- GitHub Actions CI 통과 유지

## 중요 결정사항 요약

- **딸어디가와 완전히 별개 프로젝트**
- 원래 스코프에서 잘라낸 기능: 위치 자동 추적, 반려동물, 사진일기, 투표/공지
- **Phase 1은 웹만** (브라우저 Geolocation 백그라운드 추적 불가 → 웹/앱 역할 분리)
- **Phase 2에서 React Native/Expo로 자동 지오펜싱 + 푸시 추가 예정**

## 차별점 (PRD 반영용)

1. **가족 전용 자동 공유 캘린더** — 카카오톡/일반 캘린더엔 없는, 생일·제사·기념일 자동 반복 공유
2. **표현 비용을 낮추는 프리셋 + 곱셈식 배수** — 낯간지러움 장벽 해소 + 특별한 날 의미 증폭 (예: 어버이날×2 + 엄마생신×2 = ×4)
3. **가족 공용 식물** — 개인 다마고치가 아닌 **공동 책임제**로 가족 구성원 간 자연스러운 협력 유도

## 중요 — 수동 체크인은 기술 제약이지 철학이 아님

Phase 1의 집/밖 수동 토글은 **브라우저 Geolocation 백그라운드 추적 불가라는 웹 기술 한계** 때문이며, 제품 철학(감시 반대, 수동 지향 등)이 아님. Phase 2에서는 **GPS/지오펜싱 기반 자동 체크인**으로 전환 예정. 단, 자동 전환 후에도 수동 토글은 **폴백 수단**으로 유지하여 기존 기능 보존.

→ "반자동화 철학"으로 해석하지 말 것. 면접 대응 관점에서도 "자동으로 바꿀 거면 왜 처음엔 수동이냐" 역질문 방지.

## "왜 지금" (2026 시장 맥락)

한국 가족 거주 형태 양극화:
- 성인 자녀의 1인가구 분리 증가 (떨어져 사는 가족 늘어남)
- 동시에 캥거루족 비율도 여전히 높음 (같이 살지만 생활 시간대 어긋남)

두 경우 공통적으로 "같은 집이든 떨어져 있든 가족의 존재와 마음을 확인하고 싶다"는 니즈.

## "아 이거다" 모멘트 (Activation Moment)

1. 가입 직후 가족 캘린더에 부모님 생일이 자동으로 공유되어 나타나는 놀라움
2. 프리셋 메시지 한 번으로 "사랑해/감사해요"를 보내고 부모님 반응 받은 순간
3. 가족 공용 식물이 시들기 직전 다른 멤버가 물을 주어 살린 경험
